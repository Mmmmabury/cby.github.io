网络基础知识

相关术语的中英对照

ISO: International Organization for Standards, 国际标准化组织
OSI: Open Systems Interconnection, 开放式通信系统互联参考模型
PPP：Point to Point Protocol, 点对点协议
ATM：Asynchronous Transfer Mode，异步传输模式
MAC：Media Access Control, 物理地址/介质访问控制
ISDN：Integrated Services Digital Network，综合业务数字网
B = Byte
b = bit
Mb/s = Mbps
MB/s = Mbps / 8
k 与 Ki 分别表示 kilo-（千） 与 kibi-（二进制千） 。作为前缀使用时， k 表示 1,000，Ki 表示1,024，因为“Ki”来源于它在计算机方面 210 = 1,024 的使用。不幸的是，人们常常错误地用 K 代替 Ki。此外，不知晓其中微妙差别的广大公众，常常不加区别地使用“Kbps”与“Kibps”，造成了混乱。总之，使用“Kibps”通常是正确的。

OSI参考模型中各个分层

中文名 英文名
7应用层 Application

3网络层 Network

这里要注意发送方和接收方的顺序是相反的。

会话层和传输层的区别

会话层 用来控制数据交互传输的方式，用来设定高效的传输方法没有实质的连接。
传输层 进行建立连接或断开连接的处理，是有实质的连接的。

网络层和数据链路层的区别

他们都是基于目标地址发送数据给接收端的，但网络层负责将整个数据发送给目标地址，而数据链路层值负责发送一个分段内的内容。
如果用交通作比喻的话，数据链路层就好比各种交通工具，火车，汽车，飞机等等。而网络层则好比一个目标，只关注起点和终点。

网络的构成

术语

网卡: network interface card
网络适配器:network adapter,
局域网适配器: LAN adapter
以太网适配器: Ethernet adapter

数据帧：可以理解为分组的数据

Hub：中文名集线器，为中继集线器的简称，不过现在通常指交换集线器
集线器也可以看成多口中继器。
集线器分两种：中继集线器和交换集线器

构成的硬件要素

中继器：Repeater,延长同种数据链路。
网桥/2层交换机：Bridge,连接不同的网段的设备，根据MAC地址来处理。
路由器/3层交换机：Router,连接两个不同的数据链路的设备，根据IP来处理。
4~7层交换机,例如负载均衡器
网关：Gateway,负责协议间的转换与数据的转发。

TCP/IP的标准化

严格来说，TCP/IP不是指TCP与IP这两种技术，而是指用IP进行通信时所必须的协议群的统称。

数据链路

指OSI中的数据链路层，有时也指以太网，无线局域网等。

术语

OUI：MAC厂商识别码
FDDI:光纤分布式数据接口

Preamble：前导码
FCS：Frame Check Sequence 帧检验序列
LLC:Logical Link Control 逻辑链路控制层

数据链路是什么及其作用

数据链路可以理解为：一个由通信终端和连接电路组成的系统。其中包括所用的通信协议。
数据链路是由各个节点（细致点就是网卡）和连接他们的介质，以及所用的通行协议一起组成的。

数据链路的相关技术

MAC地址（节点的标识）

6字节，48位。
MAC地址直接烧入网卡设备中，并有唯一性。
MAC地址一般由12位16进制数表示例如：01:03:13:14:25:24
二进制的MAC地址为48位

MAC地址的各个位数的含义：
第一位：表示单播地址0/多播地址1
第二位：表示全局地址0/本地地址1
第3~24位：表示厂家识别码（OUI）
第25~48位：表示厂内识别码

共享介质型网络（连接介质的使用方式）

多个节点共享一个网络介质。
基本上是半双工通信（发送与接收只能同时进行一个）
传递方式有：

  • 争用方式
  • 令牌方式

非共享介质型网络

是全双工通信：两个方向（发送与接收），可以同时进行

环路检测

在由网桥组成的带环路的网络中，需要检查是否有环路
有以下几个方法：

  • 生成树法
  • 源路由法

以太网是最为著名，使用最为广泛的数据链路。
以太网最早是共享介质型的网络通常用争用方式传递，后来发展为非共享介质。

IP 协议

术语

IP：Internet Protocol，网际协议
ICMP：Internet Control Message Protocol，因特网控制报文协议
MTU：Maximum Transmission Unit，最大传输单位
CIDR：Classless Inter-Domain Routing，无类型域间选路
VLSM：Variable-Length Subnet Mask，变长子网掩码

IP即网际协议

IP相当于OSI中的第三层（网络层）。
他的主要作用是：实现各终端节点之间的通信
而数据链路层这是在同一种数据链路中的节点间进行通信。

IP基础知识

这本书把IP大致分为三个部分：IP寻址、路由、IP分包与组包。

MAC术语数据链路层地址，而IP则属于网络层地址。
而网桥或者交换集线器等设备不需要IP地址。

路由控制（Routing）

路由控制是将分组数据发送到目标地址的功能。

  • 跳（Hop）
    跳是指网络区间。
    一跳是指用数据链路层以下的分层传输数据帧的区间。
    在以太网中，一跳就是在同一网络区间中，从一个MAC地址传输到另一个MAC地址的过程，其中不能经过路由。

  • 路由控制表（Routing Table）
    在路由器中，都有一份路由控制表。
    该表只记录与本路由相连接的路由器。也就是说只记录同一跳的路由器，例如电脑A在里两跳后的路由器B的网段中，但在本路由中，只记录在同一跳中的路由A，而不记录路由B。
    本路由————路由A————路由B————电脑A

  • 数据链路的抽象
    在IP中，数据链路是抽象化的，也就是说，IP并不涉及数据链路的具体形式。IP视所有数据链路都是相同的。他只关注头和尾。
    尽管如此，不同的数据链路依然有区别，其中最大区别就是各自的最大传输单位（MTU）。

  • IP是属于面向无连接型的
    IP在发包前不需要建立与节点的连接，而有些面向连接型的（如TCP）则，需要建立连接。
    这样的优点：

  1. 快速，效率高。
  2. 简化。
    缺点：
  3. 不可靠，会丢包等等

IP地址的基础

IP是4字节，32位的。

IP地址的分类：
A类：头8八位（第一个字节）是它的网络地址，且首位是0的地址，故范围是0.0.0.0~127.0.0.0,后24位则为主机标识。
B类：头16位（2个字节）为网络地址，前两位是10，范围为128.0.0.0到191.255.0.0
C类：网络地址为前3个字节，前三位是110，范围为192.168.0.0~239.255.255.0
D类：网络地址为所有字节，前四位为1110，范围为224.0.0.0~239.255.255.255.没有主机标识，常用于多播。

类名 网络标识 分类位 范围
A类 前1个字节 0 0.0.0.0~127.0.0.0
B类 前2个字节 10 128.0.0.0~191.255.0.0
C类 前3个字节 110 192.168.0.0~239.255.255.0
D类 前4个字节 1110 224.0.0.0~239.255.255.255

广播

同时向同一网络中得所有主机发送信息，是否接受由接受主机判断。

  • 本地广播
    网络标识一样（既同一网段上）的广播。
    广播地址一般为主机标识位都为1的IP。

  • 直接广播
    在不同网络之间的广播。

多播

可以同时向指定的主机发送信息。
多播用D类地址。

子网掩码

因为互联网发展过快，IP不够用而发展的一种新的分割IP地址的方法。
一共4字节，32位，但位为1是表示其所对应的IP的位为网络标识，若为0，则是主机标识。0与1不能交叉。

192.168.0.1
255.255.255.192

表示IP前26位为网络标识。广播地址为192.168.0.63

172.20.0.0 /16表示网络标识为16位。

CIDR与VLSM

了解一下就够。
CIDR与VLSM都是为了满足爆炸式增长的联网设备而采取的技术。
CIDR可以将不同网段的IP和为同一网段的IP，也就是放弃之前的A,B,C,D划分，可以自由的划分IP。
VLSM则是可以随机修改子网掩码的长度。

私有IP与全局IP

私有IP就是我们常说得局域网，一台主机，然后连接很多客户端时所用的IP。以前我以为私有IP是没有标准的，但是实际上在RFC上是有的标准的。

10.0.0.0 ~ 10.255.255.255 A类
172.16.0.0 ~ 172.31.255.255 B类
192.168.0.0 ~ 192.168.255.255 C类

路由控制

路由控制的核心是路由控制表（Routeing Table）
形成路由表分两种

  • 静态路由
  • 动态路由

前者是由管理员手动设置，后者是由路由器间相互交换设置。

路由控制表中记录着网路地址和下一步应该发送至的路由器的地址，并且按照最长匹配原则匹配。

路由表的一些特殊地址

  • 默认路由
    默认路由是一个任何地址都能被匹配的记录，就像万金油一样的存在。默认路由一般标记为0.0.0.0/0或default（注意区别0.0.0.0/32和0.0.0.0/0的区别，前者是IP地址0.0.0.0后者是默认路由）。

  • 主机路由
    IP地址/32的叫做主机路由（Host Route）。它基于主机地址匹配而不基于网络地址。
    一般路由表很大，效率较低

  • 环回地址
    就是127.0.0.1，也叫localhost。使用这个地址时，数据包不会流向网络。

路由表的聚合

就是利用互联网地址分层的特点将低层的地址消去，只留下高层的地址，就像树形图一样，只保留最高的父节点，而将下面的子节点删去。并且可以将各父节点相同的部分进行公示。
可以大大地减少路由表，增加效率。

IP分割处理与重组处理

为什么要有IP分割与重组呢？
因为IP的最大MTU是很大的（65535字节），而不同的数据链路，比如以太网，PPP等等的MTU是要小于IP的MTU的，所以需要进行IP分片，才能在数据链路上进行传输。
IP报文的分片以8个字节为最小单位来处理的。
约定熟成的，分片是有路由器按照下一段数据链路的MTU来处理的，然后最后在目标主机上重组，但是这样子会给路由带来非常大的压力。

路径MTU发现（Path MTU）

为了减轻路由器的分片压力，发展出了一种叫做Path MTU的技术。
路径MTU是指源主机和目标主机间，数据链路最小MTU的值。也就是说，路径MTU将原本是路由器来做的分片给源主机来处理，减少了路由器的压力。

过程：

  1. 源主机先发送第一个IP数据包，并禁止路由分片
  2. 若路由器发现该包大于路径中的MTU则丢弃，并返回路径MTU
  3. 主机按照路径MTU来分片数据包，并发送

IP协议相关技术

术语

TLD：Top Level Domain，顶级域名
DNS：Domain Name System，域名名字系统
ARP：Address Resolution Protocol，地址分辨协议
ICMP：Internet Control Message Protocol，因特尔控制报文协议

DNS

DNS将IP与域名相映射。
需要DNS的原因是因为IP与主机名不断增加，原有的集中式管理无法满足需求，所以就出现了DNS这种技术。
DNS其实可以看成是一种分布式的管理，它的服务器结构是树状的，使IP与域名的映射具有层次，易于管理。

域名

域名的出现是为了标识不同的主机，并且更加易于人类记忆。
域名的结构也是树形的，分为：

  • root（根域名）
  • 第一层域名（也叫TLD）
  • 第二层域名
  • 等等

域名服务器

就是DNS服务器，为树状；

  • root（根服务器）
  • 第一层服务器
  • 第二层服务器
  • 等等

DNS查询

解析器是用来进行DNS查询的主机或者软件。
DNS查询先是向最近DNS服务器查询，若有则返回IP，没有再向上一层进行查询
如此遍历，直到查到IP。

DNS的其他作用

DNS除了通过主机名检索IP的作用外，还有其他很多的作用。它就等同与一个互联网上的分布式数据库。

ARP

注意在IPv6中用ICMPv6代替了ARP
ARP是用来将IP和MAC地址映射的。

ARP工作机制

主机A广播ARP查询，目标主机回复自己的MAC。
ARP和DNS都有缓存，用来提高效率。

RARP

ARP的反作用。
知道MAC求IP。

ICMP

在IPv4中，ICMP的作用仅仅是辅助IP，确认网络是否正常。
ICMP的主要功能就是：

  • 确认IP包是否成功送达目标地址。
  • 通知中途废弃的IP包的原因。
  • 改善网络设置。
  • 等等

ICMPv6

ICMPv6结合了ARP，ICMP重定向，ICMP路由选择的功能。

DHCP

DHCP为网内的所有主机自动设置IP地址。
为了避免返回已使用的IP，需要做到：

  • DHCP服务器发送ICMP回送请求包，确认没有返回应答。
  • DHCP客户端针对拿到的IP发送ARP，确认没有应答。

NAT

NAT是将局域网中得私有IP在连接互联网时转为全局IP的技术。
也就是说将私有IP和外网IP映射起来。

工作机制

NAT路由将局域网过来的包得的IP地址改为外网的IP，而将外网过来的包的目标地址改为局域网中相对应的IP。

IP隧道

在网络层的首部后面继续加网络层的通信方式。主要用来满足IPv4和IPv6的同时应用。

重要的TCP和UDP

MAC对应于数据链路层的地址（标识）
IP对应于网路层的地址（标识）
而在TCP所在的传输层，它所对应的地址就是端口。也可以理解为每一个端口对应一个应用，表示数据包中的数据最后是投递到所对应的应用中。
在linux中，这些端口对应的服务器应用也叫做守护进程。
这里要注意，每一个协议所对应的端口都有不同，但是也有相同。

术语

TCP:Transmission Control Protocol
UDP:User Datagram Protocol
ACK:Positive Acknowledgement，确认应答
NACK:Negative Acknowledgement，否认应答
MSS:Maximum Segment Size，最大消息长度
Seq:Swquence Number，序列号
SYN:Synchronize Flag，连接标识
FIN:Fin Flag，断开标识

TCP和UDP的区别

他们最大的区别是可靠和不可靠。
TCP是面向有连接的协议，它具有非常多的机制来保证传输的可靠性。
UDP是面向无连接的协议，它具有高速，实时等优点。

端口号

端口号是用来区分不同应用的，例如在TCP中FTP服务器为21端口，而一个FTP客户端可能为2000端口。

通讯识别

在TCP（UDP）/IP中的通讯识别是靠：

  • 源IP地址
  • 目标IP地址
  • 协议号
  • 源端口号
  • 目标端口号

这5项来区别的，只要其中一项不一样，则表示是不同的通讯。

端口号确定

在TCP和UDP中是有标准的端口的，称为知名端口号，他们在TCP和UDP中所代表的应用都是差不多的。例如：
53号端口在TCP和UDP中都是用于DNS服务的。
但是也有只存在于一种协议的端口号。

TCP的工作机制

TCP连接和断开

  • 著名的3次握手（2个SYN，1个ACK包）
    客户端会先向服务端发送SYN包请求连接，服务端收到后返回一个SYN，最后客户端收到后再返回一个ACK来确认应答。

  • 发送FIN包断开连接
    客户端发送FIN，服务端收到后发送ACK，再发送FIN，客户端收到FIN后，发送ACK，则连接断开。

注意：先发SYN的一定是客户端，而先发FIN的不一定是客户端。

序列号（Seq）和确认应答（ACK）

序列号和确认应答可以说是TCP的基础。
序列号用于表示数据中每个字节的位置，使数据在接收端能按照原序列组合。
确认应答则是用于告诉服务端我已经接受到了之前的信息，确认应答的值是服务端上一个包的序列号+TCP中的数据长度（不包括TCP首部）。也就是说，确认应答返回给发送端一个值，这个值表示下一个包中数据的首序列。
例如：
服务端 发送数据包 Seq=1，ACK=182，Data Len=1386
客户端接收后 发送ACK Seq=182，ACK=1386+1=1387，Data Len=0

重发超时

超时都是以0.5s为单位来控制，也就是说超过0.5s的整数倍时间还没收到ACK则重发。若重发后还收不到，则时间会以指数倍延长。重发一定次数后会强制断开连接。

TCP中的MTU

TCP中得MTU叫MSS，在三次握手的时候计算得出。
TCP传输时的数据大小时靠MSS来分割的。
MSS表示TCP中的数据大小，不包括TCP首部。

TCP的窗口控制

窗口控制可以说是TCP的精髓。
窗口控制大小是指无需等待确认应答而可以继续发送数据的最大值，由接收端主机决定。
有了窗口控制后，在一定数据大小中就无需等待ACK，提高了网络效率。

窗口控制的重发是要等待收到3次确认应答才重发的。

慢启动

为了防止突然的大数据包堵塞网络，有了一个慢启动算法，对发送数据量进行控制。
