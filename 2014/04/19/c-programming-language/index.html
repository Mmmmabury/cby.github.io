类型、运算符与表达式

变量的限定符

char在c中一般是一个字节。char类型的本质是整型。

前两个满足不同的长度的整数型
short：一般为16位。短整数
long：一般为32位。长整数
unsigned：非负限定符
signed：有符号

以上四个限定符都只能限定整型（后两个包括char类型）。
另外，在ISO C99中，包含了第三种浮点数long double，对于大多数编译器来说，等价于double，理论上是用80位拓展精度来实现的。

常量

常量一般有：

  1. 整型常量
  2. 浮点数常量
  3. 字符常量
  4. 字符串常量
  5. 枚举也是种特殊常量

字符常量的一些表示

“”空字符串
‘’一个空字符
注意：”x”与’x’是不同的。’x’是一个整数，其值是对应的字符数值。而”x”则是一个包含字符x以及’\0’的字符数组。

枚举常量

枚举的的值只能在枚举内部更改，不能在外部更改。define的也一样。
要注意，枚举（enum）只能定义整型变量，而define可以定义任意的变量。

声明

变量需要声明。

int 整型 一般4字节
float 单精度 一般4字节
double 双精度 一般8字节
char 字符型 一般1字节，GBK编码时中文为2字节，而UTF-8编码时，中文为3字节

变量类别

自动变量（auto）的对立是静态变量（static）

  • 自动变量
    用堆栈机制来管理，且是一个局部变量。
  • 局部变量
    只是指那些用生命周期只在一个函数内的变量。
  • 静态变量，用static来声明
    并不用堆栈机制来使用内存，而是分配固定的内存，并且会一直占用存储空间。
  • 全局变量
    在全局都可以调用的变量。在函数外声明。
  • 寄存器变量
    存储在CPU的通用寄存器

extern 用来声明一个全局，且该变量已在其他文件中定义。
static 声明的全局变量/函数只能在本文件中使用。
const 前缀表示变量无法改变。
register 用来声明寄存器变量。

符号的种类

算术运算符：+，-，*，/，%
逻辑运算符：或||（or），与&&（and），非！
按位运算符：或运算|，与运算&，异或^，按位反~，左移<<，右移>>
取余运算%和取整只能用于int类型

类型转换

c语言中的类型转换一般是将低精度（如int）转换成高精度（如double），反过来转换也可以，不过编译器会出现警告。

注意，在转换的时候要注意变量是否是有符号的（unsigned或是signed的），若为signed，在不同机器中交换时，会发生符号的改变。

隐式转换

例如double类的与int类的进行运算（a+b）则计算机会自动将低精度的int类转为高精度double类，然后再运算。

显式转换（这个是强制转换）

例如函数change（（double）n），强制转换n为double。
注意，以上强制转换后，n的值并未改变，仅仅是生成一个指定类型的值

指针与数组

指针

指针是一种保存变量地址的变量。一般能存放2个或4个字节。且只能用于变量与数组。

一元运算符&（地址运算符）可以用于取一个对象。p = &c则p为指向c的指针。
一元运算符是间接寻址或间接引用运算符。来访问所指向的变量

注意，指针ip也是一个变量，它也有属于它自己的内存地址。
*ip的值是所指向变量的地址，而ip则是指针自己的地址

数组

一般来说，用指针来写的程序比用数组下标的程序快，但难懂。
数组用int a[10];来声明或者int a[]={x，x，......}
c中的数组可以看成是线性表。
所以若指针ip指向a[0]的话，ip++则指向数组的下一个值，（ip+1）则引用数组a[1]的值。
我们可以知道，数组名a也是代表该数组第一个变量的地址，所以将指针指向数组*第一位时我们可以是ip = &a[0]或者ip = a。注意是第一位，其他位数应该还是用第一种。

c语言进行数组运算

数组名（a）是代表该数组第一个元素的地址，而数组下标（a[i]）代表一个值。
例如c语言遇到数组a[i]时，会将其变为*（a+1）这样的形式。所以&a[i]与a+i是等价的。
数组名和指针还是有不同的。指针是一个变量，但数组名不是一个变量，它不能进行运算。
另外，在函数调用数组时，数组传递给它的只是一个自己的第一个元素的地址。
&是由变量得到地址
*是由地址找到变量

多维数组

其实多维数组能实现的功能，指针数组也能实现，只不过比较抽象和难懂。
多维数组的例子：int a[i][j]其中，i是行，j是列，所以它其实是一个行列式。

多维数组与指针数组

区别int *a[i]和int b[i][j]前者是在内存中开辟了i个n个整型的空间，其中n是动态可变的。后者是在内存中开辟了i个j个整数的空间，其中i和j都是确定的。
而且，在使用时，a[i][j]和b[i][j]都是合法的，分别是在a中第i个指针所指的数组的第j个的值，和在b中第i行第j列的值。

命令行参数

在c语言中，调用主函数main时，它带有两个参数。

  • 第一个参数（argc），表示命令行中参数的数目。
  • 第二个参数（argv[argc]），是指向字符串数组的指针，其中每个字符串代表一个参数。

按照约定，argv[0]的值是程序名，因此，argc至少为1。另外，argv[argc]是一个空指针，也就是说，最后一个可选参数的数组下标为argv[argc-1]

注意：这里就是unix命令行的一个基础，例如编译了一个c程序叫做a.out，则我们将它放入系统环境PATH路径下，然后再alias echo=“a.out”，则我们直接就可以在命令行中用echo来调用a.out程序。echo +参数例如echo hh ll后面的hh，ll就是命令行参数。

附录

枚举与define

（1）从处理过程的角度看：

#define宏是由编译预处理器在预编译处理时处理的，而且只做简单的字符串的替换。枚举常量则是在编译的时候确定其值的。

（2）从调试的角度看：

通常情况下，在编译器里，可以调试枚举常量，而不能调试宏常量。

（3）从数据的类型看：

#define可以编译任意类型的常量，而枚举只能是定义整型常量。

（4）从代码编写角度看：

枚举可以一次定义大量常量，而#define宏只能一次定义一个。

（5）从可维护性来看：

枚举可以集中管理数据，具相同属性的整形数据可使用枚举，枚举可实现取值的自增，也可指定每个枚举的值，编写代码跟容易，相对来说能减少出错的机会，也便于代码的后期维护和修改。

（6）枚举的取值范围已经限定了，容易进行参数的检查，而define没有这种检查

枚举的一些有点：可以自动赋值，可以快速的定义多个变量，可以在调试的时候显示出来，方便阅读。
