HTTP 协议是什么

我们可以先了解下网络模型。现在实际应用中的网络模型为四层：

  1. 应用层
  2. 传输层
  3. 网络层
  4. 数据链路层

越顶层越灵活，越顶层功能越多。链路层可以简单理解成网卡，网线，它在系统中对应着 MAC 地址。网络层是实现网络中各个节点的网络传输，它对应的是 IP 地址。传输层通常是指 TCP，它实现了应用程序间的可靠网络连接，对应了端口。而 HTTP 则是基于 TCP 协议的应用层协议，可以对数据进行操作。在 *nix 中一般将底下三层的技术栈封装成套接字（socket）。
一句话，HTTP（超文本传输协议）是基于应用层的协议，它的主要目的是在 Web 服务器和 Web 客户端间进行可靠的信息传输。
详情见 HTTP 历史，HTTP 标准规范，文件传输，万维网。

本要介绍什么

本文将介绍 HTTP 的基本轮廓，如果希望了解更多的细节，可以点击文中的高亮文字。文本主要包括以下部分：

  1. 通过一个简单的 Web 服务器代码，了解 HTTP 报文结构
  2. 分析 HTTP 在实际使用中会出现的问题，以及如何解决和优化，这里目前包括缓存，Cookie，session，代理和认证等。

初见 HTTP 报文

Web 客户端和 Web 服务端就像是两个邮局，而 HTTP 协议便是信使，HTTP 报文就是信使手中的包裹。
HTTP 报文分两种：

  • 客户端->服务端：请求报文
  • 服务端->客户端：响应报文

  • uri：uri 是同一资源标识符，它给互联网上的每一资源标上了唯一一个“名字”。在这里的 uri 通常代表资源在服务器里面的位置。

首部

首部的格式是根据 RFC 2822 确定，在 HTTP 协议中具体的格式是：
首部名:首部值
这样的键值对表示，并且换行符分开，在首部结束后用 CRLF(就是\r\n) 和下一部分分开。

首部也分为五部分：

  1. 通用首部。可以在请求报文以及响应报文同时使用
  2. 请求首部。请求报文特有
  3. 响应首部。响应报文特有
  4. 实体首部。用于说明实体
  5. 拓展首部。非标准的首部

其中条件请求首部、缓存相关首部、connection 首部等等会在后文有涉及。
具体的首部请参考HTTP 首部列表中文，HTTP 首部列表英文。

静态资源

静态资源是指那些不会动态改变的资源。获取静态资源很简单，像接受到一个 URI = /index.html ，说明是在主目录下的 index.html 文件，当然有些服务器可能支持虚拟 web 服务器，就是一台物理服务器含有多个 web 服务，也可以根据 host 首部进行相关设置将 URI 映射到对应的目录去。

动态资源

动态资源是指那些可以按需动态生成内容的程序，现在我们之为动态网页技术。最早的是有 CGI，后来有 FastCGI，到现在 python 中有用 WSGI。当然还有很多其他的例如微软的 ASP，还有 JSP 等等。

  • CGI
    CGI是通用网关接口（Common Gateway Interface）是一个接口标准，它定义了如何连接了 Web 服务器和外部应用。主流的 CGI 是用 Perl 写的(当然也可以用其他的)，一个 CGI 请求将调用一个 fork 来创建子进程来执行 CGI 程序。
  • FastCGI
    FastCGI(Fast Common Gateway Interface)是 CGI 的增强版本。改善了 CGI 的性能。
  • WSGI
    Web服务器网关接口（Web Server Gateway Interface，缩写为WSGI）是为Python语言定义的Web服务器和Web应用程序或框架之间的一种简单而通用的接口。

具体的CGI、FastCGI 和 WSGI 区别请看这里。
